# データベースモデル

## 概要
データをどのような構造で管理するかを定義したモデル。歴史的な変遷と、それぞれの得意・不得意がある。

## 理解したこと

### 代表的なデータベースモデルの比較

| モデル | 構造 | メリット | デメリット |
|--------|------|----------|------------|
| 階層型DB | ツリー（親子） | 親→子アクセスが高速 | 多対多の関係が表現しにくい |
| リレーショナルDB（RDB） | 2次元表（テーブル） | ACID特性による高い整合性 | 大量データのJOINが重い・スキーマ定義が必要 |
| オブジェクト指向DB | オブジェクト | プログラミング言語と直結 | 特定言語への依存・普及しなかった |
| ドキュメント型（NoSQL） | JSON/BSON（スキーマレス） | 大量データの高速処理・変換不要 | 整合性の厳密な維持が苦手 |

### リレーショナルDBが主流になった理由

| 要因 | 内容 |
|------|------|
| 標準言語の存在 | SQLという共通言語があり、OODBにはそれがなかった |
| ビジネスニーズへの適合 | データを溜めて分析するビジネス用途に向いていた |
| 技術の進歩 | インデックスと処理速度の向上でJOINの重さという弱点を克服 |

### JSONの位置づけ
JSON は「データ交換フォーマット（書き方のルール）」であり DBMSではない。ドキュメント型DBがJSON/BSON形式でデータを保存するのとは区別する。
