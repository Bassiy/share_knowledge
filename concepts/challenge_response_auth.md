# チャレンジ・レスポンス認証

## 概要
「毎回違うランダムな問題（チャレンジ）」に答えることで、秘密情報を直接渡さずに認証する仕組み。

## 理解したこと
- **Challenge**：サーバーがランダムな文字列を送ってくる
- **Response**：秘密鍵でそれに署名して返す
- 毎回問題が変わるので、署名結果（レスポンス）を盗んでも次の認証には使えない
- SSH認証はこの仕組みで動いている
- PATとの違い：PATは「合言葉そのもの」を送る。盗まれたら何度でも使われる

```
盗聴者が見えるもの

HTTPS：  PAT = ghp_AbCdEfGh...   ← これ自体が鍵。終わり。
SSH：    署名結果 = xyz789...    ← 使い捨て。次は別の問題が出る。
```

- ゼロ知識証明に近い考え方：「秘密を知っていること」を、「秘密自体を渡さず」証明する
