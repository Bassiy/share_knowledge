# Secure by Default（セキュアバイデフォルト）

## 概要
ユーザーが何も設定しない状態で最もセキュアな動作をするよう設計するという原則。

## 理解したこと
- **オプトイン保護（危険）**: セキュリティは自分で有効にしないと機能しない
  - 例：VercelのsensitiveフラグはOFFがデフォルト → 知らないと守られない
- **オプトアウト保護（安全）**: セキュリティはデフォルトで有効、無効にする場合は明示的な操作が必要
  - 例：CloudflareはV8 Isolateでデフォルト全テナント分離

- ユーザーは「暗号化した = 守られた」と誤認しやすい。**暗号化とアクセス制限は別レイヤー**：
  ```
  暗号化が守るもの     → 外部の不正アクセス・DBダンプ
  アクセス制限が守るもの → 内部システム・侵害された内部ツール
  ```
- Secure by Default が守られていない設計は、ユーザーの認知ミスを誘発する（設計側の問題）
