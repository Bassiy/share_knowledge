# Claude Code コンテキスト管理

## 概要
Claude Codeのパフォーマンスはコンテキスト量に反比例する。CLAUDE.mdを最小化し、情報はエージェントが必要時に自分で参照する設計にすることで性能を維持する。

## 理解したこと
- コンテキスト（トークン数）が増えると「考える余地」が減り、Claudeの性能が落ちる
- CLAUDE.mdは100行程度に抑える。詳細情報を詰め込まない
- `@import` でファイルを読み込むと常駐トークンが増えるので使わない
- CLAUDE.mdにはファイルパスのみ記載 → エージェントが必要なときに自分で読みに行く

```
❌ 悪い例：CLAUDE.mdに全仕様・全ルールを書く（常駐トークン増大）
✅ 良い例：「詳細は docs/spec.md を参照」とパスのみ書く
```

- 各タスクの詳細は `docs/issue-xxxx/` のような作業フォルダに置く
- エージェントチームを使う場合も同様。各エージェントには必要な情報だけを渡す
