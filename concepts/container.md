# コンテナ

## 概要
ホストOSのカーネルを共有しつつ、ファイルシステム・環境変数・プロセスを独立させる軽量な仮想化技術。

## 理解したこと
- 「OSを共有してアプリ層だけ分離する」イメージ
- VMとの最大の違い：OSカーネルを持たず借りる → 軽い・起動速い（秒以下）・サイズMB単位
- 代表：Docker。複数コンテナをまとめて管理するのがKubernetes（k8s）
- 分離強度はVMより弱い（カーネルは共有のため）
- 用途：アプリの実行環境を揃える・マイクロサービス・CI/CDのビルド環境

## VMとの比較

| | VM | コンテナ |
|--|--|--|
| OSカーネル | 各自持つ | ホストと共有 |
| 起動時間 | 分オーダー | 秒以下 |
| サイズ | GB単位 | MB単位 |
| 分離強度 | 強 | 弱 |
| たとえ | 部屋ごと丸ごと借りる | 同じ家の部屋を仕切る |
