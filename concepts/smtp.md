# SMTP（Simple Mail Transfer Protocol）

## 概要
電子メールの配送に用いられるプロトコル。

## 理解したこと

### 基本
- 元々のポート番号：25番
- 現代はクラウドサービス（AWS SES・SendGridなど）が実装を担うため、SMTPコマンド詳細（HELO/EHLOなどの応答コード）の重要度は低い

### OP25BとサブミッションポートによるスパムBlock

25番ポートは認証機能がなく、直接ポートに送りつければ誰でもメールを送れた。

**OP25B（Outbound Port 25 Blocking）**
プロバイダが25番ポートを封じることで、外部へのスパム送信を遮断。
ただし正規ユーザーも巻き込む問題が発生 → サブミッションポートで解決。

**サブミッションポート（587番）**
認証機能（SMTP-AUTH）付きの窓口として用意。

```
メールアプリ → [587番：認証あり] → メールサーバ → [25番] → 受信者
```

### SMTP-AUTH
ID・パスワードによる認証。587番・465番どちらでも必須。

### ポート587 vs 465

| | Port 587（STARTTLS） | Port 465（Implicit TLS） |
|---|---|---|
| 暗号化タイミング | 途中から切り替え | 接続した瞬間から |
| 現在の立ち位置 | 最も普及・標準的 | より厳格。2018年に正式復活 |
| 認証（SMTP-AUTH） | 必要 | 必要 |

465番の歴史：1997年登場 → 1998年廃止決定 → 現場で使われ続ける → 2018年正式再推奨。
