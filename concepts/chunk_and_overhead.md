# チャンクとオーバーヘッド

## 概要
通信の文脈で頻出する2つの基礎用語。チャンクはデータの分割単位、オーバーヘッドは本来の処理以外にかかるコスト。

## 理解したこと

### チャンク（chunk）
- 大きなデータを分割した「1かたまり」のこと
- HTTPSはデータをチャンクに分割して送る設計
- チャンクが多いほど分割・ヘッダ付加・確認待ちが繰り返される

### オーバーヘッド（overhead）
- 本来やりたい処理以外にかかるコストの総称
- 例：チャンクへの分割処理、各チャンクへのヘッダ付加（カプセル化）、ACK待ち
- SSHのストリーム接続は「つなぎっぱなしで流す」のでチャンク分割のオーバーヘッドが少ない

### 比較
```
HTTPS：データ → チャンク分割 → 各チャンクにヘッダ付加 → 送信 × n回
SSH  ：データ → ストリームでそのまま流す
```
