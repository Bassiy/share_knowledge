# ネットワークインターフェース層

## 概要
TCP/IPモデルの最下位層。直接つながった隣の機器への1ホップ分の通信を担う。

## 理解したこと
- 「1ホップ」＝直接ケーブルや無線でつながった隣の機器への1区間の転送
- インターネット層（IP）が最終目的地までを管理するのに対し、この層は「次の1つ先まで」しか考えない
- 有線・無線・認証・アドレス変換と、物理的な接続に関わる役割が集まっている
- ARPはIPアドレス↔MACアドレスの変換を担い、DHCPはIPアドレスの配布を担う。役割が異なる
- RARPとDHCPは「IPアドレスがわからない端末に教える」点で似ているが、DHCPはサブネットマスク・デフォルトゲートウェイなど付随情報もまとめて配布できる上位互換


## 主要プロトコル

| プロトコル | 役割 |
|-----------|------|
| IEEE 802.3 | 有線LAN（イーサネット）の規格 |
| IEEE 802.11 | 無線LAN（Wi-Fi）の規格 |
| PPPoE | イーサネット上で1対1接続・利用者認証。プロバイダ接続で使用 |
| PPP | PPPoEの元。電話回線（ダイヤルアップ）時代のプロトコル |
| ARP | IPアドレス → MACアドレスの変換 |
| RARP | MACアドレス → IPアドレスの変換（現在はDHCPに代替） |
| DHCP | 端末へのIPアドレスの自動割り当て。RARPの後継的役割 |
