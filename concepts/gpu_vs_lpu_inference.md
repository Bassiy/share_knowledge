# GPU vs LPU：推論における設計トレードオフ

## 概要
LLM推論においてGPUは高スループットに優れ、GroqのLPUは低レイテンシに優れる。
NVIDIAはVera RubinとGroq LPUをDynamoで統合し、両方の強みを引き出す。

## 理解したこと

### GPUがレイテンシに弱い理由
- GPUは数千コアを持つが、モデルの重みはVRAM（HBM）に格納されている
- トークンを1個生成するたびに、VRAMからコアへの重み転送が発生する
- この転送待ち（メモリ帯域ボトルネック）がレイテンシの根本原因

```
GPU  : コア ←────────── VRAM   転送待ちが発生する
LPU  : コア ← SRAM             転送ステップ自体がない
```

### LPUがレイテンシに強い理由
- SRAMをコアの隣に大量搭載（Groq LPUは500MB/chip）
- メモリ転送というステップ自体が存在しない
- ただしチップあたりの総メモリ容量はGPUより小さい

### スループット vs レイテンシはトレードオフ
| | GPU | Groq LPU |
|---|---|---|
| 得意 | 大量リクエストの同時処理（スループット） | 1リクエストを超高速に返す（低遅延） |
| 弱点 | トークン1個生成が遅い | チップあたりの総メモリが少ない |
| 向いてる用途 | バッチ推論・学習 | リアルタイム対話・音声AI |

### Dynamoによる統合（Vera Rubin + Groq LPU）
LLM推論を2フェーズに分けて担当を振り分ける：

- **Prefill**（プロンプト全体を一括処理）→ GPU担当
  - 「今日の天気を教えて」→ 文脈を一気に並列処理
- **Generation**（トークンを1個ずつ逐次生成）→ LPU担当
  - 「今」「日」「は」...と低遅延で出力

DynamoがリクエストをみてGPU/LPUを自動振り分けする。

### ラックスケールコンピューティング（Vera Rubin）
- 5ラックをNVLink（260TB/s）で接続し、ソフトウェア的には「1台の巨大マシン」として扱える
- 分散処理の複雑さをハードウェア側が吸収し、開発者は意識しなくてよくなる
- 設置2時間・100%液冷で「インフラとして量産・展開できる」基盤へ
