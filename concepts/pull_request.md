# Pull Request

## 概要
コードの変更をチームに確認・承認してもらうための申請の仕組み。

## 理解したこと
- 自分のブランチでの変更を「本番コードに取り込んでいい？」と申請するもの
- GitHubではコードの差分（diff）が表示され、行単位でコメントをつけられる
- チームメンバーが確認・承認したらマージ（本番コードに統合）される
- AIコードレビューはこのPRへの自動コメント投稿として実装されることが多い
