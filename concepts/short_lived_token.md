# 短命トークン（Short-lived Token）

## 概要
必要なときだけ発行され、短時間で自動失効する使い捨ての認証情報。

## 理解したこと
- .envに書く永続的なAPIキーと対比される概念
- 有効期限が1時間以内など短く、漏洩しても放置すれば無効になる
- AWSではSTS（Security Token Service）が一時的な認証情報を発行する
- 「漏らさない設計（SSH鍵など）」ではなく「漏れても大丈夫な設計」という発想の転換
- 呼び出されるまでパスワードは存在しない → 金庫の鍵を管理会社から必要なときだけ借りるイメージ
- 短命トークン + IPアドレス制限 + 最小権限スコープ の3点セットで被害を最小化

## 権限借用（Impersonation）との関係
- 権限借用は「短命トークンをどうやって取得するか」の手段
- サービスアカウントAがサービスアカウントBになりすまし、Bの権限で一時トークンを発行する仕組み
- GCPではJSONキー（永続・固定パスワード）の代わりに権限借用を使うのがベストプラクティス
- 「鍵を渡す」のではなく「都度通行証を発行する」イメージ

## 混同ポイント
- **SSH鍵認証と混同しやすい**（どちらも「安全な認証」という印象）
- 根本的な違いは**鍵の有無**：
  - SSH鍵認証 → 暗号鍵ペアが存在する。秘密鍵で署名するから漏れない設計
  - 短命トークン → 鍵はない。ただの文字列だが短時間で自動失効するから漏れても大丈夫な設計
