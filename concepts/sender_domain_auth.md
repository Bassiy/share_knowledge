# 送信ドメイン認証（SPF / DKIM / DMARC）

## 概要
メールの送信元なりすましを防ぐための認証技術群。SMTP-AUTHだけでは送信者アドレスの偽装は防げないため、これらが必要。

## 理解したこと

### 3つの技術

| 技術 | 役割 |
|------|------|
| SPF | 「このドメインのメールはこのIPからしか送らない」というリストをDNSに公開。受信側がIPを照合して正当性を確認する |
| DKIM | メールヘッダに電子署名を付与。改ざん検出と送信元の正当性を証明する |
| DMARC | SPF/DKIM失敗時のポリシーを定義（拒否・迷惑メール行きなど） |

### 3段構えのイメージ
- SPF：「どこから送っていいか」を宣言
- DKIM：「本当にそこから来たか」を証明
- DMARC：「ダメだったらどうするか」を決める
