# 責務分離（Separation of Concerns）

## 概要
1つのものが担う仕事を1つに絞る設計の考え方。

## 理解したこと
- 「なんでも屋」を作らない。役割を明確に分けると、変更・デバッグ・保守が楽になる
- 何かが壊れたとき「どこが原因か」がすぐ絞れる
- 1つを変更したとき他に影響しない範囲が明確になる
- 個人開発でも有効：未来の自分が困らないために「境界線を意識しながら書く」だけでも違う
- Railsテストでの具体例：
  - Request Spec → HTTPの契約（ステータス・レスポンス形式・認証）だけ
  - Query Spec   → DBのデータ取得ロジックだけ
  - Model Spec   → ビジネスルールだけ
- AI設計でも同じ：スキル（手順を持つ）とSubagent（実行する）を分ける

## 関連概念
- solid_principles（SRPは責務分離を原則として定式化したもの）
- ai_subagent（専門Subagentへの分割も同じ発想）
