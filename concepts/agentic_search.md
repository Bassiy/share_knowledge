# Agentic Search

## 概要
「エージェントが自律的に検索する」という概念だが、実装方針によって2種類の定義がある。

## 理解したこと

### 定義A：Claude Code 方式（grep/glob ベース）
- 「Agentic Searchとはglobとgrepのことだ。それだけ」（Claude Code開発者 Boris Cherny）
- RAGが不要になった理由：LLMが曖昧な表現から適切な検索キーワードを自分で生成できるようになったから
- エージェントは「ヒットしなければ別のキーワードで再試行」「ディレクトリ構成から推測」もできる
- 前提条件：ディレクトリ構成・命名規則が整理されていること、CLAUDE.mdで案内があること
- このプロジェクトの search-sources や recall スキルも同じ思想（直接grep設計）

### 定義B：反復RAGループ方式
- LLMエージェントが「計画→検索→評価→再検索」を自律的に繰り返すアーキテクチャ
- 従来RAGの「クエリ固定・1ショット」の限界を超える
- 三層構造：エージェント層（思考・判断）/ ツール層（ベクトル検索・API）/ データ層（複数ソース）
- トレードオフ：精度は上がるがレイテンシー・コスト・デバッグ難度が増す
- 適用判断：複雑な多段階推論・複数ソース横断・精度最優先のケースで検討する

### RAGとの比較（定義A vs 定義B）
- 定義A（grep）はRAGを**置き換える**。構造化データ向け
- 定義B（反復RAG）はRAGを**拡張する**。非構造化・不特定多数向け
