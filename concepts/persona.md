# ペルソナ

## 概要
典型的なユーザー像を架空の人物として具体化したもの。マーケティング・UX設計の用語。

## 理解したこと
- 本来は実際のユーザーインタビューや調査データをもとに作るもの
- 属性情報（年齢・職業・性別）だけでなく、**心理的特性（思考の癖・価値観・行動パターン）**まで書くことで精度が上がる
- 「あの人ならどう言いそうか」を予測するとき、職業より思考の癖の方が予測精度が高い

### 属性だけ vs 心理まで書く
| 浅いペルソナ | 深いペルソナ |
|--|--|
| 34歳、マーケター | 34歳、マーケター、KPIで物事を測る癖がある、技術コストを常にROIで換算する |
| → 「効率が大事だと思います」（当たり前） | → 「月8回制限なら実装コスト1週間、解約率3%改善で回収できます」（具体的） |

### AI活用との組み合わせ
- Claude Code Agent Teamsで各ペルソナを担当するエージェントに「演じさせる」ことができる
- 1プロンプト + ペルソナ定義でユーザーリサーチの仮説を高速生成できる（30分で優先度付きロードマップが出た事例あり）
- **位置づけ**：実際のインタビューの代替ではなく、仮説を立てる前段階として機能する
