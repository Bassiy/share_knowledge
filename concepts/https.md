# HTTPS（HyperText Transfer Protocol Secure）

## 概要
HTTP通信をSSL/TLSによって安全化したプロトコル。「HTTP + TLS」と表現できる。

## 理解したこと

### 基本
- ポート番号：443番
- HTTPのシンプルさ・汎用性を損なわずに安全性を追加できている点が革新的
- 実質的にはTLSが使われている（SSLは現在非推奨）

### データの流れ（送信時）
1. HTTP → 生データを生成
2. TLS → 暗号化
3. TCP → セグメント分割、ポート番号・シーケンス番号を付与

受信時は逆順。

### カプセル化によって実現

```
┌─────────────────────────────────────────┐
│  アプリケーション層         HTTP         │  生データを渡すだけ
├─────────────────────────────────────────┤
│            TLS（暗号化・復号）            │  ← ここに挟み込まれる
├─────────────────────────────────────────┤
│  トランスポート層           TCP          │  セグメントを運ぶだけ
└─────────────────────────────────────────┘
```

各層は隣のレイヤーとやり取りしているつもりのまま動くため、HTTPもTCPも改造不要。
