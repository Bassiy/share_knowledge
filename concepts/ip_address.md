# IPアドレス

## 概要
IPプロトコルを使うネットワーク上で、各コンピュータを識別するために付与される番号列。

## 理解したこと
- IPv4 と IPv6 の2種類がある
- IPアドレスは「デバイスそのもの」ではなく「ネットワーク上の住所」に付与される
- 同じデバイスでもネットワークが変わればIPアドレスは変わる（自宅WiFi → カフェWiFiなど）
- 固定IPでない限り、再接続のたびに変わることもある（動的割り当て）
- デバイス固有の識別子はMACアドレス（製造時に焼き付けられる）。IPアドレスとは別物

### グローバルIPアドレス vs プライベートIPアドレス

| | グローバルIP | プライベートIP |
|---|---|---|
| 範囲 | インターネット全体 | 内部ネットワーク内 |
| 一意性 | 世界全体で重複不可 | 同一ネットワーク内で重複不可（別ネットワーク間はOK） |
| 用途 | インターネット上の通信 | LAN内の通信 |

### IPアドレスの構造

```
IPアドレス = ネットワーク部（所属ネットワークの特定）＋ ホスト部（端末の特定）
```

例）クラスC の場合（ネットワーク部24bit / ホスト部8bit）

```
192 . 168 . 1 . 10
|_____________| |__|
  ネットワーク部  ホスト部
  （192.168.1）   （10）
```

| 用語 | 具体例 | ニュアンス |
|------|--------|-----------|
| IPアドレス | `192.168.1.10` | 端末のフルネーム |
| ネットワーク部 | `192.168.1` | 「前半24ビット分」という範囲のこと |
| ネットワークアドレス | `192.168.1.0` | 一番最初。グループ名そのもの |
| ブロードキャストアドレス | `192.168.1.255` | 一番最後。全員への一斉放送用 |

> ネットワークアドレス（先頭）とブロードキャストアドレス（末尾）の２つがネットワーク内のホスト部を最初から占有（システムが予約）しているため、端末が使える特定ネットワーク内でのIPアドレスは２引かれる。

### クラス

先頭ビットのパターンでクラスが決まる。実用上はA〜Cを使う。

| クラス | ネットワーク部 | 先頭ビット | 用途 |
|-------|-------------|----------|------|
| A | 8bit | `0` | 大規模ネットワーク |
| B | 16bit | `10` | 中規模ネットワーク |
| C | 24bit | `110` | 小規模ネットワーク |
| D | — | `1110` | マルチキャスト用（特殊） |
| E | — | `1111` | 予約済み（特殊） |

- ネットワーク部が広い（A）ほど収容できる端末数が多い
- ネットワーク部が狭い（C）ほど作れるネットワーク数が多い

### ネットマスク

- IPアドレスとネットマスクをAND演算するとネットワークアドレスが得られる
- ネットマスクはホスト部を全て0にするためのビット列

**管理体系**
```
ICANN（総本山）
└── 地域インターネットレジストリ（例：APNIC）
    └── 国別インターネットレジストリ（例：JPNIC）
        └── ローカルインターネットレジストリ（例：ISP）
            └── ユーザへの割り当て
```
