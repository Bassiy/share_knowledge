# AI Design to Figma

## 概要
Claude CodeとFigma MCPを接続し、AIがデザインシステムに準拠したUIを自動生成してFigmaへ直接出力するワークフロー。

## 理解したこと

### フロー
```
要件をClaudeに伝える
    ↓
Claude Code + Figma MCP
    ↓ デザインシステム（Figmaライブラリ）を参照
Reactコードを生成
    ↓
Figmaへ自動出力
```

### AIが得意なこと・苦手なこと
- **得意**: デザインシステムに準拠した一貫性のあるコンポーネント生成・量産
- **苦手**: ビジネス要件の解釈、UXの本質的な改善提案、優先順位の判断

### デザイナーの役割シフト
- 制作者（Figmaでコンポーネントを作る人）
- → 戦略顧問（何を作るべきか決める人・AIの判断を評価する人）

### 限界
- Figmaは見た目の確認のみ。インタラクション・レスポンシブ・状態変化は確認できない
- 最終的にコードへの変換が別途必要（2ステップになる）
- Figmaが有効なのはデザイナーとのハンドオフや非エンジニアへのレビューが必要な場面
